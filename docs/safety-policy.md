# Safety Policy

## Product Promise

`GO TO WORK` exists to help users automate approved local work, not to hide control from them.

## Policy Rules

- all automation is local and user-visible
- approvals are explicit and revocable
- sensitive actions receive stronger scrutiny
- desktop control is fallback-only
- no bypassing account or platform security
- no stealth monitoring
- no credential harvesting
- no arbitrary file access features

## Enforcement Mechanisms

- allowlists
- pending approvals
- timed leases
- visible indicator
- emergency stop
- action logs
- screenshot evidence where practical

## Operator Responsibilities

Users should:
- keep allowlists narrow
- avoid approving sensitive actions blindly
- prefer browser workflows over desktop fallback
- review audit output during testing and before release packaging

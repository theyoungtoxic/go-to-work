# Branding Boundaries

## Positioning Rule

`GO TO WORK` must be marketed as an independent third-party automation product.

## Do

- say it is MCP-compatible
- say it works with AI agents and clients
- describe integrations factually
- make it clear the user runs it locally under their own control

## Do Not

- imply official affiliation with OpenAI, Anthropic, GitHub, Microsoft, or browser vendors
- use partner or certified language without a real agreement
- present logos or naming that suggests endorsement
- imply hidden agent autonomy beyond user-approved control

## Product Copy Direction

Preferred language:
- `works with MCP-compatible AI clients`
- `local browser and desktop automation under user approval`
- `third-party Windows automation control plane`
